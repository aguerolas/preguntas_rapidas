# executive

El diagrama representa la arquitectura objetivo del sistema biométrico de Scotiabank Perú, es decir, cómo debería quedar el sistema una vez implementado completamente. El flujo empieza en las sucursales, donde el oficial bancario usa una workstation conectada a un escáner de huellas. Cuando un cliente necesita ser verificado, la huella se captura y viaja hacia el servidor central de forma segura.
Ese servidor central, alojado en el datacenter LATAM de BGOV, corre sobre Linux (RHEL 8.9) con Tomcat 10 y tiene dos instancias separadas: una para CSF y otra para SBP. Cada una tiene sus propios servicios web desplegados como archivos .war, lo que significa que operan de forma independiente. Esto es importante porque un problema en una instancia no debería afectar a la otra.
Las huellas capturadas se validan contra RENIEC a través de una conexión TLS, y también se consultan los sistemas core del banco. Los datos biométricos se almacenan en SQL Server 2019, también on-prem, con bases de datos separadas por entorno: una para CSF, una para SBP y una general de biometría.
Adicionalmente, el sistema se integra con FastBank, iSTS, un módulo de seguridad que incluye un servicio de inteligencia artificial, y un sistema de reportería. Las credenciales de conexión a la base de datos se gestionan a través de una herramienta llamada Cryptotool, lo que agrega una capa extra de seguridad en el manejo de accesos.
